# Инструкция по заполнению JSON-файлов для PRAVA

## 1. Общие принципы
- Один JSON = одна статья (или определение) с вопросами и переводами.
- Все языки (FR, NL, RU) хранятся в одном файле.
- NL берётся с wegcode.be, RU — машинный или ручной перевод.
- Вопросы (exam_questions) делаются на каждое определение, минимум 3 на определение.
- К каждому вопросу можно добавить промпт для генерации изображения.
- SEO-метаданные обязательны для статьи и каждого определения/вопроса.
- Все поля, даже если пустые, должны присутствовать для валидации.

## 2. Структура файла
- `category` — объект с названием раздела, slug, описаниями на 3 языках.
- `article_number`, `title`, `title_nl`, `title_ru` — заголовки статьи.
- `seo` — SEO-метаданные для статьи (title, description, keywords на 3 языках).
- `content_html`, `content_html_nl`, `content_html_ru` — HTML-контент.
- `content_text`, `content_text_nl`, `content_text_ru` — чистый текст.
- `definitions` — массив определений (см. ниже).
- `images` — массив изображений (знаки, схемы, промпты).
- `order`, `source_id` — для сортировки и идентификации.
- `_meta` — служебные поля (источник, статус, версия, переводчик).

## 3. Структура определения (definitions[])
- `id` — уникальный идентификатор (например, "2.3").
- `term`, `term_nl`, `term_ru` — термин на 3 языках.
- `text`, `text_nl`, `text_ru` — определение.
- `signs` — массив кодов знаков.
- `cross_references` — массив ссылок на другие статьи.
- `seo` — SEO-описание на 3 языках.
- `exam_questions` — массив вопросов (см. ниже).

## 4. Структура вопроса (exam_questions[])
- `text`, `text_nl`, `text_ru` — формулировка вопроса.
- `difficulty` — сложность (1-3).
- `image` — объект с кодом знака и промптом для генерации.
- `options` — массив вариантов (A/B/C, тексты на 3 языках, is_correct).
- `explanation`, `explanation_nl`, `explanation_ru` — объяснение.

## 5. Структура изображения (images[])
- `sign_code` — код знака.
- `alt_text`, `alt_text_nl`, `alt_text_ru` — alt-текст.
- `caption` — подпись.
- `source_url` — ссылка на оригинал.
- `generation_prompt` — промпт для генерации.
- `order` — порядок.

## 6. _meta
- `source_type` — pdf|scrape|manual
- `source_url` — ссылка на источник
- `import_version` — версия шаблона
- `last_updated` — дата
- `validation_status` — draft|reviewed|approved
- `translated_by` — gpt-4o|human|deepl

## 7. Валидация
- Использовать _schema.json для проверки структуры перед импортом.
- Все поля должны быть заполнены или пустыми (но присутствовать).

# Инструкция по генерации и импорту
1. Скрапить/парсить статью с официального сайта (FR, NL).
2. Перевести на русский (LLM или вручную).
3. Сгенерировать минимум 3 вопроса на определение (FR, NL, RU).
4. Добавить промпты для генерации изображений.
5. Заполнить SEO-метаданные.
6. Проверить структуру через _schema.json.
7. Импортировать через management-команду.
8. Проверить и доработать в админке (картинки, ревью, статус).
